7关键技术决策

7.1 数据流选择

  • ✅ 采用：HTML 内嵌同步定义 9 节课程
  • ❌ 拒绝：纯异步加载（导致"课程空白"问题）
  • 📋 原则：核心展示数据必须 inline 在 HTML 中，确保用户即时可见

7.2 师承图谱设计

  • ✅ 采用：横向时间轴 + 节点连接（适配移动端 overflow-x）
  • ✅ 采用：金色边框（命理）+ 青色边框（风水）视觉分流
  • ❌ 拒绝：复杂 D3 拓扑图（学习成本高、移动端不友好）

7.3 速查表维度

  • ✅ 采用：3 维度（流派 / 名言 / 著作），共 26 条
  • ✅ 采用：auto-fill 网格 + 名言左侧色条引用块
  • 📋 依据：直接复用 KB 中已有 quick_reference 与 lineage_overview 数据结构

7.4 Tab 设计

  • ✅ 采用：4 → 6 tab（增 🌳 师承图谱 + 📖 速查表）
  • ✅ 采用：tab 顺序按"用户学习路径"：舒晗 → 倪师 → 历代宗师 → 师承 → 速查 → 学习路径
