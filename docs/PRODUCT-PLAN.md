## 一、产品概述

### 1.1 产品定位
乾元命理宝鉴公众号H5是一个集国学工具、知识学习、修行实践、养生调理、开运商城于一体的综合型国学文化平台。

### 1.2 目标用户
- 对国学命理感兴趣的微信用户
- 希望学习传统文化的缘主
- 有信仰修行需求的信众
- 注重养生健康的人群

### 1.3 核心价值
- **工具引流**: 12种命理工具吸引流量
- **内容留存**: 知识体系+修行体系提升留存
- **会员变现**: 4级会员体系+积分体系实现商业化
- **反馈优化**: AI评估用户建议，持续改进产品

## 三、会员体系设计

### 3.1 会员等级

| 等级 | 价格 | 权益 |
|------|------|------|
| 免费用户 | ¥0 | 每工具限用1次/天 |
| 月度会员 | ¥9.9/月 | 不限次数+基础报告 |
| 年度会员 | ¥99/年 | 不限次数+深度报告+商城9折 |
| 终身会员 | ¥299 | 不限次数+全部功能+商城8折+1对1咨询 |

### 3.2 积分体系

| 行为 | 积分 |
|------|------|
| 每日签到 | +5 |
| 完成测评 | +2 |
| 分享好友 | +10 |
| 提交有效建议 | +20~100 (AI评估) |

- 100积分 = 1元，可兑换会员

### 3.3 使用限制逻辑
1. 用户首次使用需验证手机号
2. localStorage记录手机号+每日使用次数
3. 免费工具: 非会员每天1次，会员不限
4. 付费工具: 仅会员可用
5. 达到限制弹出会员引导Modal

## 四、建议反馈+AI价值评估

### 4.1 用户端流程
1. 用户填写: 建议类型(功能/内容/Bug/其他) + 建议内容 + 期望改进
2. AI自动评估 (前端实现):
   - 基础分: 30分
   - 有建设性(+15): 内容>30字
   - 详细描述(+10): 内容>80字
   - 指出具体问题(+15): 含数字/操作描述
   - 可操作(+10): 有期望改进
   - Bug类加分(+10)
   - 影响面大(+10): 涉及会员/支付/登录
3. 评估结果: 1-100分 + 对应积分奖励
4. 80分以上: 标记为高价值建议，可减免会员费

### 4.2 管理端流程
1. 后台建议反馈管理页面查看所有建议
2. 按状态(待处理/已采纳/已拒绝)和类型筛选
3. 查看AI评估分数和奖励积分
4. 操作: 采纳/拒绝 + 积分奖励发放

## 五、后台管理系统 (admin.html)

### 5.1 模块架构 (9个模块)

| 模块 | 功能 |
|------|------|
| 📊 运营大屏 | 实时数据/功能热度/转化漏斗/用户画像/收入统计/行为路径 |
| 👥 用户管理 | 用户列表/试用次数/会员开通续费退款 |
| 💬 建议反馈 | 建议列表/AI评分/采纳拒绝/积分发放 |
| 🔧 引擎测试台 | 排盘引擎调试(原有) |
| 📚 知识库管理 | 知识卡片统计/搜索/导出(原有) |
| 🧎 信众指导中心 | 日课/参拜/时辰/节气/经文/禁忌(原有) |
| 📝 内容管理 | 口诀/语录/生肖(原有)+商品/Banner(扩展) |
| 📈 数据统计 | 功能使用/付费转化/用户留存 |
| 📊 系统监控 | 服务状态/文件统计(原有) |

### 5.2 数据来源
- H5页面通过localStorage('mlbj_admin')记录用户行为事件
- 后台读取同一localStorage展示数据
- 模拟数据填充展示效果
- 后续可对接后端API

### 5.3 运营大屏功能
- **实时数据**: 在线人数/今日访问(PV/UV)/今日测评次数/今日收入
- **功能热度排行**: 柱状图展示各工具使用次数
- **会员转化漏斗**: 访问→试用→付费的转化率
- **用户画像**: 性别分布/年龄段分布
- **收入统计**: 日/周/月收入报表
- **热门测评**: 最常使用的测评类型排行
- **用户行为路径**: 入口→行为→转化的路径分析
