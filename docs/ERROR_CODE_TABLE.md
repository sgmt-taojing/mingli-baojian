# 错误码完整表

> 自动维护（来源：server/api-response.js）

| 错误码 | 含义 | HTTP | 用户文案 |
|--------|------|------|---------|
| 0 | 成功 | 200 | - |
| 400001 | 参数校验失败 | 400 | 请检查输入内容 |
| 401001 | 未登录 | 401 | 请先登录 |
| 401002 | Token 过期 | 401 | 登录已过期 |
| 403001 | 无权限 | 403 | 您没有访问权限 |
| 404001 | 资源不存在 | 404 | 内容不存在或已被删除 |
| 409001 | 资源冲突 | 409 | 操作冲突，请刷新后重试 |
| 429001 | 全局限流 | 429 | 请求过于频繁 |
| 429002 | KB 限流 | 429 | 知识库调用过快 |
| 500001 | 服务端异常 | 500 | 服务异常，请稍后再试 |
| 503001 | AI 不可用 | 503 | AI 暂时不可用，已切换知识库 |
| 503002 | DB 不可用 | 503 | 数据服务维护中 |

## 扩展指南

新增错误码必须：
1. 在 `server/api-response.js` 的 `ERROR_CODES` 注册
2. 在 `docs/ERROR_COPYWRITING.md` 加用户文案
3. 在前端 `app/js/api-client.js` 的 `ERROR_HANDLERS` 加处理动作
