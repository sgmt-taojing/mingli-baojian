3.2 五层加密架构

加密层次模型
层级名称措施
L5应用层加密（字段级）病患姓名/生辰八字/症状/方剂/大师分析均AES-256-GCM加密，密钥由KMS管理
L4传输层加密HTTPS(TLS 1.3) + HSTS + API请求HMAC-SHA256签名
L3数据库加密（TDE）openGauss透明数据加密，全库at-rest加密，密钥90天轮换
L2存储加密对象存储SSE-S3，KB文件靠权限隔离（非加密），日志脱敏
L1密钥管理主密钥KMS管理永不落盘，DEK每表/每租户独立，DEK 90天轮换，Master Key 365天轮换

四、大师诊断结论单独存放

普通病例 (medical_cases)

· 病患自助填写
· 简单症状记录
· 无命理分析
· 医生常规诊疗
· localStorage（当前）
· 安全级别: L3
· 保留: 365天

大师案例库 (master_cases)

· 大师亲自分析
· 完整八字+起卦+医学化翻译
· 医生审核流程
· 数据库加密存储（AES-256-GCM）
· 支持知识蒸馏
· 安全级别: L4
· 保留: 5年（医疗+研究价值）

案例审核流程

草稿 (draft) ← 大师创建/编辑案例
待审核 (submitted) ← 大师提交，医生收到通知
审核中 (reviewing) ← 医生查看完整案例
退回修改 (revision_requested) ← 医生要求修改 → 大师修改后重新提交
已通过 (approved) ← 医生通过 → 最终方案生成
已完成 (completed) ← 推送报告给病患
已归档 (archived) ← 保留期满/主动归档

五、知识蒸馏提取机制

五步蒸馏流程

1
案例收集
触发条件：每月/每季度 / 新增高质量案例≥50条 / 检测到未覆盖新模式。筛选标准：status=completed + quality_score≥7 + effectiveness≥3
2
模式提取
特征提取（五行/脏腑/卦象/体质/证候/方剂）→ 按(五行+体质+证候)分组聚类 → 统计有效率 → 置信度计算（阈值≥0.6，支持案例≥5，矛盾比例≤20%）
3
知识验证
自动验证（交叉比对+逻辑一致性+方剂匹配+去重85%）→ 人工审核（大师审周易 + 医生审中医 + 管理员审分类）→ 三方均通过才入库
4
KB更新
匹配目标KB文件（白名单7个）→ 新增/补充/修正 → 每条标注来源案例ID → 安全检查（无个人信息泄露）→ 生成diff
5
版本管理
版本号MAJOR.MINOR.PATCH（MINOR级递增）→ 快照存储+diff记录 → 管理员确认后生效 → 支持回滚到任意历史版本
