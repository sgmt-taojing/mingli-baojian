一、四层专业分级体系

平台按用户专业程度和服务深度分为 4 层，每层有独立的报告生成逻辑、引导深度和完整报告输出。

T1AI 助手 · 大众层

面向零专业知识用户。用友好对话全量引导，引导完毕给出完整报告。

  • 问题引导：一步步问，不假设用户懂术语
  • 引导深度：8-15 题，覆盖基础信息 + 核心关切
  • 报告：AI 助手全量报告（命理概要 + 建议 + 行动清单）
  • 端：H5 + 小程序 + 微信 H5 均可
核心 URL：
ai-assistant.html — AI 全引导 + 报告
my-yuanzhu.html — 我的助手/推送/积分
yuanzhu-inbox.html — 消息收件箱
后端：/api/ai/chat · /api/ai/guide

T2排盘引擎 · 爱好者层

面向有学习兴趣或专业知识的用户。8 大引擎排盘后给出完整专业报告。

  • 八字 · 紫微 · 奇门 · 大六壬 · 六爻 · 梅花 · 风水 · 易卦
  • 用户可自选引擎、自定义参数
  • 报告：结构化命盘 + 十宫分析 + 流年推演 + 化解建议
  • 端：H5 大屏 + 小程序 + 微信 H5

T3易+医 · 高端层

面向高端客户。命理 + 中医双轨并行，AI + 真人医师/大师联合服务。

  • 排盘诊断 → 体质辨识 → 中医方案 → 跟踪随访
  • AI 辅助 + 大师/医师审核确认
  • 报告：命理报告 + 中医诊疗方案（电子病历）

三、智能眼镜集成方案（新需求）

🥽智能眼镜 · AI 辅助对话引导系统

智能眼镜作为AI 数据采集入口，实时采集视频/音频 → 后端 AI 分析 → 指导授权角色（周易大师/医师）与信众对话 → 对话完毕自动形成方案 → 审核后推送。

3.1 数据流

1

眼镜采集

智能眼镜实时采集视频 + 音频流，通过 WebSocket 或 RTSP 推送到后端 /api/glass/stream

2

AI 实时分析

后端 AI 引擎（g2claw / GLM-4V）分析视频内容 + 语音转文字 → 提取关键信息 + 对话上下文

3

智能提示

AI 生成对话引导建议（3-5 条），实时推送到授权角色（大师/医师）的眼镜 HUD 或手机端

4

对话执行

授权角色按引导与信众对话，采集信息，眼镜全程录制

5

方案自动生成

对话完毕，AI 自动生成完整方案（命理报告 + 中医建议），写入待审核队列

6

审核 + 推送

大师/医师审核确认 → 推送至信众/患者 H5 + 小程序 + IM

3.4 数据库表

CREATE TABLE glass_sessions (
  id INTEGER PRIMARY KEY,
  master_id INTEGER,           -- 授权角色
  patient_id INTEGER,          -- 信众/患者
  stream_url TEXT,             -- 音视频流地址
  transcript TEXT,             -- 对话转写
  ai_analysis TEXT,            -- AI 分析 JSON
  guide_suggestions TEXT,      -- 引导建议 JSON
  auto_report TEXT,            -- 自动生成的方案草稿
  status TEXT DEFAULT 'active', -- active/ended/approved/rejected
  created_at TEXT DEFAULT (datetime('now','localtime'))
);

四、三端全量报告逻辑

T1

AI 助手报告流程（大众层）

输入：自然语言问题
引导：AI 友好提问（8-15 题，覆盖基础信息 + 关切点）
引擎：KB 优先双路径（命中≥0.7 直答，0.4-0.7 AI+KB 润色，<0.4 AI 兜底）
输出：完整报告（基础信息 / 现状分析 / 优势 / 风险 / 建议 / 时机 / 化解 / 总结 + 行动清单）
端点：/api/ai/chat · /api/ai/guide

T2

排盘报告流程（爱好者层）

输入：生辰八字 + 选引擎
排盘：8 大引擎专业排盘（八字/紫微/奇门/六壬/六爻/梅花/风水/易卦）
分析：引擎输出命盘 + 十宫分析 + 流年推演
输出：完整专业报告（命盘 / 格局 / 大运 / 流年 / 化解建议）
端点：/api/paipan/*

T3

易+医报告流程（高端层）

输入：排盘结果 + 症状描述 + AI 面相/舌苔（智能眼镜）
诊断：命理分析 + 中医辨证（八纲/六经/三焦）
方案：命理建议 + 中医处方 + 食疗 + 穴位 + 经方
审核：AI 草稿 → 大师/医师审核确认
输出：双轨报告（命理 + 中医）
端点：/api/clinic/* · /api/face/* · /api/glass/report/*

T4

后台报告流程（专业层）

输入：全平台数据汇总
审计：运营数据 / 排盘质量 / 推送效果 / KB 命中率
输出：管理报告（KPI 仪表盘 + 异常告警 + 数据导出）
端点：/api/admin/* · /api/monitor/*
