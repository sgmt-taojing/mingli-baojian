## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | 现有HTML/JS/CSS | 保留现有，渐进式升级 |
| 后端API | Node.js Express | 轻量高性能 |
| 数据库 | openGauss（华为开源） | 兼容PostgreSQL，免费 |
| 缓存 | Redis | 排盘结果缓存 |
| 安全 | JWT+AES+HTTPS | 加密传输+脱敏存储 |
| 部署 | Docker+nginx | 容器化部署 |
| 监控 | 日志+告警 | 异常预警 |

## 数据库表设计（openGauss）

```
users          用户表（手机号加密/生辰加密/VIP/信仰偏好）
paipan_records 排盘记录（八字/奇门/紫微等排盘结果）
orders         订单表（商城购买/分成记录）
merchants      商家表（道观寺庙/大师入驻）
feedback       反馈表（点赞/点踩/建议/积分）
knowledge      知识库表（命理/风水/养生/咒语）
courses        课程表（名师课堂视频/音频/文字）
push_logs      推送日志（每日推送记录/打开率）
audit_logs     操作日志（安全审计）
system_config  系统配置（分成比例/推送策略）
```

## 安全设计

```
传输层：HTTPS全站加密
存储层：手机号/生辰AES-256加密
应用层：JWT鉴权+RBAC权限
防护层：防SQL注入/XSS/CSRF/暴力破解
审计层：全操作日志
```

## 并发设计

```
单机阶段（0-500用户）：
  Mac mini M4 16GB
  nginx + Node.js + openGauss + Redis

扩容阶段（500-5000用户）：
  + 阿里云2核4G（API服务器）
  + OSS对象存储（视频/图片）
  Mac mini跑数据库+开发

大规模（5000-10000+用户）：
  + 多台API服务器负载均衡
  + CDN加速静态资源
  + 数据库读写分离
  + 提醒用户上云
```

## 风险与对策

| 风险 | 对策 |
|------|------|
| 硬件不足 | 用户已计划扩展硬盘+内存 |
| API Key暴露 | 后端代理转发，前端不直接调用 |
| 数据丢失 | openGauss自动备份+日志 |
| 并发瓶颈 | Redis缓存+连接池+限流 |
| 知识错误 | 回测验证+用户反馈+名师审核 |
