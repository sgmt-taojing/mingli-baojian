2. 两次反馈的真正根因（挖掘）

根因 A：语音助手无法用（重犯）

❌ 表面现象： 浏览器 Console 提示 window.VoiceInteraction 未定义
❌ 深层根因：
  1. ai-assistant.html 第 151 行 <script src="voice-interaction.js"> — 文件实际在 app/js/voice-interaction.js，HTTP 404，整个脚本未加载
  2. voice-interaction.js 是 IIFE 闭包，即便加载成功也只 export window.startVoiceInput window.speakText 等单函数
  3. ai-assistant.html 写的是 window.VoiceInteraction.startListening() — 桥接对象从不存在
  4. 降级路径直接 fallback 到原生 SpeechRecognition，但 iOS Safari 没有，给的提示是浏览器不支持
✅ 修复方案：
  1. script src 改为 js/voice-interaction.js?v=20260721（与 divination-hub 一致）
  2. 新增 window.VoiceInteraction = { startListening, stopListening, isListening, speak, stop, isPlaying, setVoice, stopAll } 桥接
  3. ai-assistant.html 现有 vi.isListening() / vi.startListening() 全部生效

根因 B：拍照还是走文件选择器

❌ 表面现象： iOS Safari 点按"📷 拍照"弹的是"照片图库"或"文件选择器"
❌ 深层根因：
  1. <input style="display:none"> + <button onclick="input.click()"> 模式在 iOS 上不可靠
  2. WebKit 把 button click 触发的 click() 视为 programmatic click on hidden element → 直接走文件选择器
✅ 修复方案：
  1. display:none → position:absolute; width:1px; height:1px; opacity:0; clip:rect(0,0,0,0)（视觉隐藏但 iOS 可识别）
  2. <button onclick> → <label for="inputId">（label click 是浏览器原生 trusted event，最可靠触发相机）
📍 涉及文件： ai-assistant.html（faceUpload）+ tcm-clinic.html（症状/大师/医生 3 组拍照按钮）
