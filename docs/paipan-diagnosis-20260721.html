5. 婚姻引导断链 · 根因分析

🔴 根因 #1 · MODULES 没有独立 marriage 入口
用户点击 AI 助手中的"婚姻/正缘"按钮时，期望进入 6-8 步逐步引导流程（年龄 → 感情史 → 当前状态 → 关注点 → 时间窗 → 化解方案），但实际 MODULES 中不存在'marriage'键。用户只能走 bazi 流程并在第 4 步选"感情婚姻"，得到的仅是一段嵌入八字报告的文字。
修复：在 ai-assistant.html 的 MODULES 对象中新增 'marriage':{name:'正缘指引',icon:'💕',desc:'婚姻情感诊断',steps:[…7步…]}，并在引导菜单加按钮
🔴 根因 #2 · bazi 流程中"感情婚姻"分支未实际分支
bazi 模块第 4 步选项包含"感情婚姻"，但 generateReport() 没有根据该选项做真正的分支处理，仍是生成通用八字综合报告。用户在 bazi 流程里选"感情婚姻"得到的报告里婚姻段可能被淹没在五行十神长文里。
修复：generateReport() 根据 state.data.s3（关注方面）做 focus 模式，单独生成婚姻专项报告；或新建独立 marriage 模块
🟡 根因 #3 · _MODULE_REPORTS 兜底可能缺失
generateReport 优先调用 _MODULE_REPORTS[module].diagnose() 离线兜底。如果用户在断网场景下点击婚姻引导，且 _MODULE_REPORTS 没有对应模块，会直接降级到 /api/ai/public-chat 调用失败 → 报告空白。
修复：新建 marriage 模块时同步加入 _MODULE_REPORTS.marriage = {name:'婚姻情感KB兜底', diagnose(data){…}}
🟡 根因 #4 · 排盘结果容器 ID 缺失
排盘结果写入目标如 #paipanResult / #baziResult 在 hub 中 未定义，意味着 computeBazi() 等函数执行后找不到写入位置，结果可能直接 console.log 后丢失。
修复：补全 divination-hub.html 中所有结果容器的 DOM 占位
🟡 根因 #5 · onclick handler 6 个未定义
event / document / this / showFengshuiProSub / fsProCompute / koujueToggleDetail 这 6 个 onclick 调用找不到对应函数。event/document/this 极可能是 dev 残留，需要清理。
修复：grep -nE "onclick=\"(event|document|this)\\(" divination-hub.html，定位后删除或修复
