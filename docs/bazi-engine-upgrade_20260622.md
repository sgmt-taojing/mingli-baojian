### 第三轮：辅助命理函数 + 前端对接

#### 1. 长生十二宫 `getDishi(gan, zhi)` — ✅ 已验证
- 阳干顺行、阴干逆行，从 `CHANGSHENG_START` 起算
- 4个用例 × 4地支 = 16项全部正确

#### 2. 旬空 `getXunKong(dayStem, dayBranch)` + `getXunName()` — ✅ 已验证
- 旧算法：`(stemIdx - branchIdx + 12) % 10` → 错误
- 新算法：在60甲子中找位置 `n`，`xunIdx = n//10`，空亡 = `(xunIdx*10+10)%12` 和 `+11`
- 返回空亡地支 + 旬名（甲子/甲戌/甲申/甲午/甲辰/甲寅）

#### 3. 命宫 `getMingGong(yearStemIdx, monthBranch, timeBranch)` — ✅ 已验证
- 旧算法：只用月支逆数到卯（缺少时支参数，结果错误）
- 新算法（对标 lunar_python）：
  - 月支、时支均在 `MONTH_ZHI_ARR`（1-based, 寅=1）中查索引
  - `offset = monthIdx + timeIdx`
  - `if offset >= 14: offset = 26 - offset; else: offset = 14 - offset`
  - 天干：`(yearStemIdx + 1) * 2 + offset`，超10减10

#### 4. 身宫 `getShenGong(yearStemIdx, monthBranch, timeBranch)` — ✅ 已验证
- 旧算法：只用月支顺数到酉（错误）
- 新算法（对标 lunar_python）：
  - 月支用 `MONTH_ZHI_ARR` 查，时支用 `ZHI_ARR`（1-based, 子=1）查
  - `offset = monthIdx + timeIdx`
  - `if offset > 12: offset -= 12`
  - 天干同命宫

#### 5. `getBaziCalcData` 全面升级
返回对象新增字段：
- `mingType`：旺衰诊断对象（strengthLevel, tongScore, yiScore, ratio, yongshenEle, jishenEle, tiaohou, isCongGe, ...）
- `geju`：格局对象（name, source, desc）
- `shensha`：神煞数组（name, desc）
- `tenGods`：十神数组
- `taiYuan` / `mingGong` / `shenGong`：辅助命理柱
- `xunKong` / `xunName`：旬空地支 + 旬名
- `dishi`：长生十二宫数组（4个地支各一）
- `dayun`：大运数组

#### 6. 前端显示对接
- 命盘速览面板新增：胎元/命宫/身宫/旬空/旬名/长生十二宫
- 核心指标卡片升级：3列→6列（日主/格局/用神/旺衰/忌神/同党占比）
